# P2-03-1：极简历史摘要压缩

- **status**: pending
- **改进项**: #18 会话摘要与记忆压缩（极简版）
- **前置任务**: P1-02-1
- **后续任务**: P2-07

## 目标

当对话历史过长时，不再机械地只取最近固定条数消息，而是采用**极简压缩方案**：

- 较早历史消息先临时压成一段 `summary`
- 最近少量消息保留原文
- 最终把 `summary + recent raw messages + session state + 当前用户输入` 一起送入模型

本任务的目标是**压缩消息**，不是引入完整 memory 系统。

## 当前代码

`packages/server/src/services/ai/index.ts` 中：

- `loadHistory(userId, limit = 20)` 只会读取最近 20 条历史消息
- 更早消息直接不进入本轮上下文
- 当前没有 `summary`
- 当前也没有“旧历史压缩 + 最近少量原文”的混合上下文模式

这会导致两个问题：

1. 早期仍然有用的历史事实会被直接截断
2. 最近 20 条中可能混有重复追问或低价值文本，未必都是最值得保留的上下文

## 本任务明确不做

为了保持实现简单，V1 **不做以下内容**：

- 不新增摘要数据表
- 不持久化 `summary`
- 不引入 `summarizedUntilMessageId`
- 不做滚动增量摘要
- 不做长期记忆 / user profile

也就是说，V1 的 `summary` 是**请求内临时生成、仅本轮使用**的上下文压缩结果。

## 具体改动

### 1. 当历史过长时，临时压缩旧消息

在加载完历史消息后，增加一个简单判断：

- 如果历史消息较短，保持现状，直接使用原始历史
- 如果历史消息超过阈值，则：
  - 较早消息 → 先生成一段临时 `summary`
  - 最近少量消息 → 保留原文

建议的 V1 默认值：

```typescript
const SUMMARY_TRIGGER_MESSAGE_COUNT = 12;
const RECENT_RAW_MESSAGE_COUNT = 8;
```

即：

- 历史消息不超过 12 条：不压缩
- 历史消息超过 12 条：只保留最近 8 条原文，其余较早消息做临时摘要

### 2. `summary` 只总结历史事实，不承担流程状态

压缩时要明确边界：

- `summary` 只保留仍然可能影响后续对话的历史事实
- 不负责记录当前确认态 / 挂起动作

也就是说：

- “用户最近主要在安排什么事”
- “已经确认过哪些任务事实”

适合进 `summary`

而：

- “当前正在等待删除确认”
- “这句‘确认’要恢复哪个动作”

不应主要靠 `summary` 承担，而应继续由 `session state` 负责

### 3. 采用极简摘要 Prompt

V1 使用一个很轻的摘要策略即可，不需要复杂 schema。

摘要目标建议写清楚：

- 只总结历史事实
- 只保留后面可能还会用到的信息
- 不复述完整聊天过程
- 不写当前确认态

输出形式建议为短段落或 3-5 条简短要点，方便直接注入上下文。

### 4. 上下文结构改为“摘要 + 最近原文”

本任务完成后，单 Agent 的上下文组装思路应从：

```typescript
system prompt + 最近 20 条历史 + 当前用户输入
```

变为：

```typescript
system prompt + session state + summary(如有) + 最近 8 条原始消息 + 当前用户输入
```

如果本轮没有触发摘要，则不注入 `summary`。

### 5. 摘要失败时必须降级

如果临时摘要生成失败：

- 不阻塞主链路
- 直接降级为仅使用最近少量原始消息

也就是说，本任务属于“提高上下文质量”的增强项，不应成为聊天主流程的新故障点。

## 涉及文件

- `packages/server/src/services/ai/index.ts`（或拆分后的 `history-manager.ts` / `context builder`）
  - 历史消息分段逻辑
  - 临时摘要生成逻辑
  - 上下文组装逻辑

## 验收标准

- [ ] 历史消息较短时，行为与当前实现保持一致
- [ ] 历史消息过长时，不再机械使用最近固定 20 条
- [ ] 旧历史可被临时压成 `summary`，最近少量消息保留原文
- [ ] `summary` 不承担确认态 / 挂起动作这类流程状态
- [ ] 摘要失败时可降级，不影响主链路回复
